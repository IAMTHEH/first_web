    웹호스팅(github pages)

    직접 웹서버를 운영하는 일은 쉽지 않을 일입니다.
    그래서 이런 일을 대행해 주는 회사들이 있습니다.
    인터넷에 연결된 컴퓨터 하나 하나를 호스트라고 하고 이런 컴퓨터를 빌려주는 사업을 호스팅이라고 합니다.
    우리는 수많은 웹호스팅 업체 중 무료이고, 유명한 서비스인 github(https://github.com)을 이용할 것입니다.
    github는 수많은 오픈소스들이 만들어지는 곳이기 때문에 프로그래머들에게는 성지와 같은 중요한 서비스 입니다.
    우선 github에 회원가입하시고 로그인 한 다음 소스코드를 보관하는 저장소(repository)라는 것을 생성해야 합니다.
    new repository 라는 버튼을 누르면 다음과 같은 화면이 나오는데 저장소를 생성하는 화면입니다.
    출처 : https://opentutorials.org/course/3084/18891
    1. Repository name 에는 프로젝트의 이름을 적어 주세요. 저의 경우 web1이라고 할 것입니다.
    2. public(공개), private(비공개)는 저장소의 공개여부를 정하는 것입니다.
    public으로 해 주세요. (누구나 여러분의 소스코드를 볼 수 있습니다)
    3. Initialize this repository with a README의 체크박스를 꼭 체크해 주세요.
    그 다음 저장소 생성하기(Create repository)버튼을 누르면 다음와 같은 화면이 나타날 것입니다.

    출처 : https://opentutorials.org/course/3084/18891
    이 중에서 upload files라는 부분을 클릭하면 다음과 같은 화면이 나타날 것입니다.
    출처 : https://opentutorials.org/course/3084/18891
    1. choose your files(파일을 선택하세요)을 누르세요. 프로젝트 폴더에 있는 파일을 다중선택합니다.
    2. 변경된 내용을 적습니다. 파일이 변경될 때마다 업로드를 해야 하기 때문에 변경된 내용을 적어줍니다.
    3. Commit changes 버튼을 누르세요. 업로드가 시작됩니다.
    업로드가 끝나면 파일의 목록이 보입니다.
    이제 우리가 업로드한 웹페이지를 인터넷을 통해서 서비스 해봅시다. Settings 버튼을 선택합니다. pages 항목에서 branch:main를 선택하고 Save 버튼을 누릅니다.
